Microservices Architecture
🛍️
Product Service
Manage product catalog (Pizza, Drinks, Bread); add / update / remove products; stock controlled by Supplier
📦
Order Service
Order placement, tracking, history; parallel queue; auto-cancel unpaid orders after 10 min
🗃️
Inventory Service
Track availability; auto-update stock on confirmation; low-stock notifications to Supplier
💳
Payment Service
Dummy QR (order_id + amount); 'I have completed payment' button; auto-cancel if not clicked in 2 min
🔔
Notification Service
Order confirmations, status updates; alerts on order placed, fulfilled, cancelled, or payment failure
🎁
Promotions & Offers
Seasonal / festive discounts, coupons, scheduled promos; weekend specials, health-day & hourly offers

Backend — Spring Boot Maven Project
📁 Root
├─ 📁 src/main/java/com/hackathon/retail/
│ ├─ ☕ SmartRetailApplication.java Main entry point
📦 Entity Layer
├─ 📁 entity/
├─ ☕ User.java @Entity — id, name, email, password, role(ENUM: CUSTOMER|SUPPLIER)
├─ ☕ Product.java @Entity — name, category, price, stock_qty, is_available, supplier(FK)
├─ ☕ Order.java @Entity — customer(FK), status(ENUM), total_amount, created_at, paid_at
├─ ☕ OrderItem.java @Entity — order(FK), product(FK), quantity, unit_price
├─ ☕ Payment.java @Entity — order(FK), qr_code, amount, status(ENUM), confirmed_at
├─ ☕ Promotion.java @Entity — name, discount_pct, coupon_code, valid_from, valid_to, type(ENUM)
└─ ☕ Notification.java @Entity — user(FK), order(FK), message, sent_at, is_read
🗄️ Repository Layer
├─ 📁 repository/
├─ ☕ UserRepository.java findByEmail, findByRole
├─ ☕ ProductRepository.java findByCategory, findBySupplierId, findByIsAvailable
├─ ☕ OrderRepository.java findByCustomerId, findByStatus, @Query day-wise supplier view
├─ ☕ OrderItemRepository.java findByOrderId
├─ ☕ PaymentRepository.java findByOrderId, findByStatus
├─ ☕ PromotionRepository.java findActiveBetween, findByCouponCode
└─ ☕ NotificationRepository.java findByUserId, findUnreadByUserId

├─ ☕ ProductService.java CRUD products, manage availability
